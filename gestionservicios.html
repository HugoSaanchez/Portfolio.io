Gestión de Servicios para Alumnos y Empresas

Este proyecto consiste en desarrollar una plataforma web de gestión de servicios, ofrecidos por alumnos de distintas familias profesionales del centro educativo. La plataforma gestiona tres tipos de usuarios: alumnos, empresas y administradores.

Alumnos: Estudiantes de diferentes especialidades (informática, electricidad, soldadura, etc.), que se registran para realizar servicios solicitados por empresas.
Empresas: Solicitan servicios y, una vez realizados, dejan comentarios y valoraciones sobre los alumnos que completaron el trabajo.
Administradores: Son responsables de gestionar a los alumnos, las empresas y los servicios solicitados, así como asignar alumnos a los servicios de acuerdo a sus habilidades profesionales.

Funcionalidades principales:

  • Registro de usuarios: Alumnos, empresas y administradores se registran en la plataforma con diferentes roles y permisos según sus funciones.
  • Gestión de servicios: Las empresas pueden solicitar servicios relacionados con diferentes familias profesionales (informática, electricidad, soldadura, etc.).
  • Asignación de servicios: Los administradores asignan servicios a los alumnos en función de sus competencias, y se establece una fecha para la realización del servicio.
  • Comentarios y valoraciones: Las empresas pueden comentar y valorar los servicios realizados por los alumnos, lo que promueve la retroalimentación y mejora continua.
  • Generación de informes: Al finalizar un servicio, el alumno genera un informe detallado que documenta su trabajo y los resultados obtenidos.

Herramientas y Tecnologías:

Backend:

  • Desarrollado en Java utilizando Spring Boot para la lógica del servidor.
  • Base de datos gestionada con MySQL para almacenar la información de usuarios, servicios y valoraciones.
  • Control de versiones y colaboración en GitHub.
  • Despliegue realizado en la plataforma web Render.

Frontend:

  • Desarrollado con HTML, CSS y JavaScript para la interfaz de usuario.
  • Uso de Thymeleaf como motor de plantillas para generar vistas dinámicas en el servidor.
  • Integración con el backend mediante API REST desarrollada en Spring Boot.
